Emulation: Defines the basic surface emulation strategy, with the following options:


"None" - the program controls the desktop color depth.

"Primary Buffer" - same as "None," but blit operations are made against a memory surface and then transferred to the real primary surface – this handles the otherwise known pitch-bug problem.

"Locked Surface" -

"Primary Surface" - the virtual primary has a different color depth of the real primary, and DxWnd takes care of the color transformation internally: slower but no screen mode changes!

"Automatic mode" - uses a very simple strategy for Direct3D games: it tries the "Primary Surface" mode first and, if that doesn't succeed, switches to "Locked Surface." It is the best strategy I came across, but it doesn't always work!
